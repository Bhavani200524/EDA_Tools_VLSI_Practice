Synchronous fifo
________________

External ports that fifo will require
	clk_i, rst_i, wr_en_i, wdata_i, rd_en_i, rdata_o, full_o, empty_o, error_o

	_i indicates it is input to the design. 
	_o indicates it is output of the design.

	error_o is raised only when we try to write into a full fifo or
	when we read from an empty fifo

Two ways to write logic for synchronous fifo
	implementation 1:
	we put data one after the other on same location
		here fifo wil have to make sure, once the data is written, it is shifted to reading address
	from the other end after reading out one data element, 
		entire buffer structure is shifted so that next element is read out in correct order.
	
	implementation 2:
	use pointers to read and write data
	pointers cannot be externally available. 
	they need to be internal logic
		while implementing them we need to think about num of locations available in fifo
		pointer width that is required to point to all locations
	
	At reset, both wr_ptr and rd_ptr will point at 0th location
	fifo is empty at this point
	when i write 1 data element, it will write it at 0th location and wr_ptr will immediately increment to 1
	wr_ptr always points to location which needs to be written next
	when i write 1 data element, it will write it at 1st location and wr_ptr will immediately increment to 2
	at this point since i have not read out anything, rd_ptr points to 0th location
	when i read from 0th location, read pointer immediately increments to 1
	rd_ptr will always point at next in line data element to be read.

How to conclude fifo is full or empty
		i have written to my fifo till 5th location 
		wr_ptr at this point is pointing 6th location wr_ptr=6
		i have read out of the fifo till 3rd location 
		rd_ptr at this point is pointing 4th location rd_ptr=4

		at this point, 0th to 3rd location are empty and 6th to 15th locations are also empty
		but fifo structure is not empty because we can still read out 2 data. 4th and 5th
		we cannot even say fifo is full

		whenever wr_ptr and rd_ptr do not match, we will find, fifo is neither empty nor full

		write to all 16 locations wr_ptr=0
		read from 13 locations rd_ptr=13
		fifo neither empty nor full

		write to all 16 locations wr_ptr=0
		read from 0 locations rd_ptr=0
		fifo is full

		write to 0 locations wr_ptr=0
		read from 0 locations rd_ptr=0
		fifo is empty

		write to all 16 locations wr_ptr=0
		read from 7 locations rd_ptr=7
		write to 7 locations wr_ptr=7
		fifo is full

		we need one more flag variable to indicate whether fifo is full/empty
		
		fifo_wr_flag=0
		fifo_rd_flag=0

		Let's say, after writing to all 16 locations, everytime wr_ptr comes back to 0 from 15
		fifo_wr_flag=1
		rd_ptr=0
		fifo_rd_flag=0
		at this point fifo is full.	

		now I will read out of all 16 locations rd_ptr will comes back to 0 from 15
		fifo_rd_flag=1
		wr_ptr=0
		fifo_wr_flag=1
		at this point fifo is empty


		pointer value equal will indicate fifo is either empty or full
			here we need to check flag value to decide whether empty or full
		pointer value not equal will indicate fifo is neither empty nor full
			no action required

What if we don't use pointers?
	there will be no track of order in which data is put
	we won't know if fifo is full/empty data loss possible
	
What if we don't use full and empty signals?
	data loss
	overriting
	reading out incorrect data

What if we don't use toggle flags?
	not possible to calculate full/empty condition
	
Why a particular width of pointer variable?
	depending on num of locations required to access
	in our case, 16 num of locations 
	so 4 bit pointer is enough to address all

External ports that fifo will require
	clk_i, rst_i, wr_en_i, wdata_i, rd_en_i, rdata_o, full_o, empty_o, error_o

What are main interfaces of fifo?
	2 main interfaces:
		wr_interface
			wr_en_i, wdata_i, full_o
		rd_interface
			rd_en_i, rdata_o, empty_o
	common signals to both interfaces:
		clk_i, rst_i, error_o

	
Internal registers:
	wr_ptr, rd_ptr, wr_toggle_f, rd_toggle_f

Memory structure
	reg[Width-1:0]fifo[Depth-1:0]
