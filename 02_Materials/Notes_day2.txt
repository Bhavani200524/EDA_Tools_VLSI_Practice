always block w.r.t wr clk
	only write operation will be performed here
	write error will be updated here
always block w.r.t rd clk
	only read operation will be performed here
	read error will be updated here

wr_ptr, rd_ptr, 
wr_toggle_f, rd_toggle_f

internal variables of write operation are updated wrt write clock
internal variables of read operation are updated wrt read clock

while calculating full and empty conditions, they are not really comparable

We will need to calculate wr_ptr_rd_clk this value wrt rd clock
we can compare this with rd_ptr
to calculate empty condition, we will use comparison of wr_ptr_rd_clk and rd_ptr

We will need to calculate rd_ptr_wr_clk this value wrt wr clock
we can compare this with wr_ptr
to calculate full condition, we will use comparison of wr_ptr and rd_ptr_wr_clk

to generate full and empty conditions, 
	we got the solution of synchronizing the ptr and flag values wrt other clock
	this solution had one issue of too many bits changing at the same time in synchronizer
	this issue won't be seen is simulation 
	but it will be seen in real hardware
	to resolve this issue we used gray code
		we can convert binary ptr value to gray value in corresponding always block
		or we can convert it in synchronizer stage

	2nd option will leave us with same issue of too many bits being changed
	we can convert binary ptr value to gray value in corresponding always block

to write run.sh
	because there are many commands that we need to give for a particular simulation
	otherwise same commands can be executed in terminal separately
	run.sh
		vcs -full64 -sverilog -debug_access+r -kdb <file_name.v>
		./simv
		#above will execute code in terminal itself. signal changes during execution are saved in provided file.
		./simv -gui &
		#above  will open verdi tool with verdi console
		with run command in verdi console you will be able to execute code in the tool
	if you are writing run.sh file change its permissions to executable
	chmod +x run.sh command
	you can then directly give
	./run.sh command in terminal
	to open verdi in debug mode
	verdi -ssf fsdb_file_name.fsdb
