verilog is hardware description language
	most constructs from verilog synthesizable
but for verification we don't need to have synthesizable constructs

sv was developed to make verification easier
	it is hardware verification language
	all the verilog constructs are present as it is in sv
	a lot of verfication related constructs are added. 
	it is easier to make tb modular and resusable with sv.
	modular means compartmentalised: specific job perfromed by specific component
	eg. job of stimulus generation : gen will take care of stimulus generation
	job of driving stimulus to the dut: bus functional module bfm/driver
	in case of issues in tb, it is easier to narrow it down to specific component and debug.
	in complex examples, we see having separate components performing separate jobs makes it easy to add more testcases or analysis processes

	reusable means, once i have IP for a particular protocol written, I don't have to write it again and again. I can simply use the data driving environment in my current design if it uses that communication protocol.
